Création d'un middleware personnalisé pour gérer l'accès
propriétaire de manière propre et compatible Laravel 11.


═══════════════════════════════════════════════════════════════
📂 FICHIERS CRÉÉS/MODIFIÉS
═══════════════════════════════════════════════════════════════

1️⃣  NOUVEAU MIDDLEWARE
   ────────────────────────────────────────────────────────────
   Fichier : app/Http/Middleware/EnsureUserIsOwner.php
   
   Ce middleware :
   • Vérifie que l'utilisateur est connecté
   • Vérifie que l'utilisateur a le rôle 'owner'
   • Redirige vers login si non connecté
   • Retourne 403 si pas propriétaire


2️⃣  ENREGISTREMENT DU MIDDLEWARE
   ────────────────────────────────────────────────────────────
   Fichier : bootstrap/app.php
   
   Ajout de l'alias 'owner' pour le middleware


3️⃣  APPLICATION DANS LES ROUTES
   ────────────────────────────────────────────────────────────
   Fichier : routes/web.php
   
   Ajout de ->middleware('owner') sur le groupe admin


4️⃣  SIMPLIFICATION DES CONTRÔLEURS
   ────────────────────────────────────────────────────────────
   Fichiers :
   • app/Http/Controllers/AdminController.php
   • app/Http/Controllers/UserController.php
   
   Suppression de toutes les vérifications manuelles
   Le middleware s'occupe de tout !


5️⃣  SCRIPT DE NETTOYAGE
   ────────────────────────────────────────────────────────────
   Fichier : NETTOYER_CACHE.bat
   
   Pour nettoyer tous les caches Laravel


═══════════════════════════════════════════════════════════════
